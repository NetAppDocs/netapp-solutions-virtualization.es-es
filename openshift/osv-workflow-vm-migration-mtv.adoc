= Migrar una máquina virtual de VMware a un clúster de Red Hat OpenShift
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Migra máquinas virtuales de VMware a un clúster de OpenShift usando el kit de herramientas de migración de virtualización de OpenShift. Esta migración implica instalar el Migration Toolkit for Virtualization (MTV), crear proveedores de origen y destino, crear un plan de migración y realizar una migración en frío o en caliente.

.Migración en frío
[%collapsible%open]
====
Este es el tipo de migración predeterminado. Las máquinas virtuales de origen se apagan mientras se copian los datos.

====
.Migración cálida
[%collapsible%open]
====
En este tipo de migración, la mayoría de los datos se copian durante la etapa de precopia mientras las máquinas virtuales (VM) de origen están en ejecución.  Luego, las máquinas virtuales se apagan y los datos restantes se copian durante la etapa de transferencia.

====

== Migración de máquinas virtuales de VMware a OpenShift Virtualization mediante el kit de herramientas de migración para virtualización

Esta sección describe cómo usar Migration Toolkit for Virtualization (MTV) para migrar máquinas virtuales desde VMware a OpenShift Virtualization ejecutándose en OpenShift Container Platform e integrado con almacenamiento NetApp ONTAP usando Trident.

El siguiente diagrama muestra a grandes rasgos la migración de una máquina virtual de VMware a Red Hat OpenShift Virtualization.

image:rh-os-n-use-case-vm-migration-using-mtv.png["Figura que muestra el diálogo de entrada/salida o representa contenido escrito"]



[[prerequisites-for-the-sample-migration]]
=== Requisitos previos para la migración de muestra



[[on-vmware]]
=== En VMware

* Se instaló una VM RHEL 9 usando RHEL 9.3 con las siguientes configuraciones:
+
** CPU: 2, Memoria: 20 GB, Disco duro: 20 GB
** Credenciales de usuario: usuario raíz y usuario admin


* Después de que la máquina virtual estuvo lista, se instaló el servidor PostgreSQL.
+
** El servidor PostgreSQL se inició y se configuró para que arranque al iniciar el sistema




[source, console]
----
systemctl start postgresql.service
systemctl enable postgresql.service
----
Estos comandos garantizan que el servidor pueda iniciarse en la máquina virtual en OpenShift Virtualization después de la migración.

* Se añadieron 2 bases de datos, 1 tabla y 1 fila a la tabla. Consulta link:https://docs.redhat.com/en/documentation/red_hat_enterprise_linux/9/html/configuring_and_using_database_servers/using-postgresql_configuring-and-using-database-servers#configuring-postgresql_using-postgresql["aquí"] para ver las instrucciones sobre cómo instalar el servidor PostgreSQL en RHEL y crear entradas de base de datos y tabla.



NOTE: Asegúrate de iniciar el servidor PostgreSQL y de habilitar el servicio para que se inicie al arrancar.



[[on-openshift-cluster]]
=== En OpenShift Cluster

Las siguientes instalaciones se completaron antes de instalar MTV:

* OpenShift Cluster 4.17 o posterior
* Rutas múltiples en los nodos del clúster habilitadas para iSCSI (para la clase de almacenamiento ontap-san).  La multi-ruta se puede habilitar fácilmente si instala Trident 25.02 usando el indicador node-prep.  Puedes consultar ellink:osv-trident-install.html["Sección de instalación del Trident"] Para más detalles.
* Instale las clases de almacenamiento y backend requeridas y la clase de instantánea.  Consulte lalink:osv-trident-install.html["Sección de instalación del Trident"] Para más detalles.
* link:https://docs.redhat.com/en/documentation/openshift_container_platform/4.19#Virtualization["Virtualización OpenShift"]




[[install-mtv]]
=== Instalar MTV

Ahora ya puedes instalar el Migration Toolkit for Virtualization (MTV). Consulta las instrucciones proporcionadas link:https://access.redhat.com/documentation/en-us/migration_toolkit_for_virtualization/2.5/html/installing_and_using_the_migration_toolkit_for_virtualization/installing-the-operator["aquí"] para obtener ayuda con la instalación.

La interfaz de usuario de Migration Toolkit for Virtualization (MTV) está integrada en la consola web de OpenShift. Puedes consultar link:https://access.redhat.com/documentation/en-us/migration_toolkit_for_virtualization/2.5/html/installing_and_using_the_migration_toolkit_for_virtualization/migrating-vms-web-console#mtv-ui_mtv["aquí"] para empezar a usar la interfaz de usuario en varias tareas.



==== Crear proveedor de origen

Para migrar la máquina virtual RHEL de VMware a OpenShift Virtualization, primero debes crear el proveedor de origen para VMware. Consulta las instrucciones link:https://access.redhat.com/documentation/en-us/migration_toolkit_for_virtualization/2.5/html/installing_and_using_the_migration_toolkit_for_virtualization/migrating-vms-web-console#adding-providers["aquí"] para crear el proveedor de origen.

Necesita lo siguiente para crear su proveedor de origen de VMware:

* URL de vCenter
* Credenciales de vCenter
* vCenter huella digital del servidor
* Imagen VDDK en un repositorio


Ejemplo de creación de un proveedor de origen:

image:rh-os-n-use-case-vm-migration-source-provider.png["Figura que muestra el diálogo de entrada/salida o representa contenido escrito"]


NOTE: El kit de herramientas de migración para virtualización (MTV) utiliza el SDK del kit de desarrollo de discos virtuales (VDDK) de VMware para acelerar la transferencia de discos virtuales desde VMware vSphere.  Por lo tanto, crear una imagen VDDK, aunque opcional, es muy recomendable.  Para utilizar esta función, descargue el VMware Virtual Disk Development Kit (VDDK), cree una imagen VDDK y envíe la imagen VDDK a su registro de imágenes.

Siga las instrucciones proporcionadaslink:https://access.redhat.com/documentation/en-us/migration_toolkit_for_virtualization/2.5/html/installing_and_using_the_migration_toolkit_for_virtualization/prerequisites#creating-vddk-image_mtv["aquí"] para crear y enviar la imagen VDDK a un registro accesible desde el clúster OpenShift.



==== Crear proveedor de destino

El clúster de host se añade automáticamente como proveedor de destino.



==== Crear un plan de migración

Siga las instrucciones proporcionadaslink:https://access.redhat.com/documentation/en-us/migration_toolkit_for_virtualization/2.5/html/installing_and_using_the_migration_toolkit_for_virtualization/migrating-vms-web-console#creating-migration-plan_mtv["aquí"] para crear un plan de migración.

Al crear un plan, debe crear lo siguiente si aún no lo ha hecho:

* Un mapeo de red para mapear la red de origen a la red de destino.
* Una asignación de almacenamiento para vincular el almacén de datos de origen con la clase de almacenamiento de destino. Para esto, puedes elegir la clase de almacenamiento ontap-san. Una vez creado el plan de migración, el estado del plan debería mostrar *Ready* y ahora deberías poder *Start* el plan.


image:rh-os-n-use-case-vm-migration-mtv-plan-ready.png["Figura que muestra el diálogo de entrada/salida o representa contenido escrito"]



[[perform-cold-migration]]
=== Realizar migración en frío

Haz clic en *Iniciar* para seguir una serie de pasos y completar la migración de la VM.

image:rh-os-n-use-case-vm-migration-mtv-plan-complete.png["Figura que muestra el diálogo de entrada/salida o representa contenido escrito"]

Una vez completados todos los pasos, podrás ver las máquinas virtuales migradas haciendo clic en *Máquinas virtuales*, dentro de *Virtualización*, en el menú de navegación de la izquierda. Para gestionar las máquinas virtuales, consulta link:https://docs.redhat.com/en/documentation/openshift_container_platform/4.22/html/virtualization/managing-vms["la documentación de OpenShift Virtualization"].

Puedes iniciar sesión en la máquina virtual y comprobar el contenido de las bases de datos de PostgreSQL. Las bases de datos, las tablas y las entradas de la tabla deberían ser las mismas que las que se crearon en la máquina virtual de origen.



[[perform-warm-migration]]
=== Realizar migración cálida

Para realizar una migración warm, después de crear un plan de migración como se muestra arriba, debes editar la configuración del plan para cambiar el tipo de migración predeterminado. Haz clic en el icono de edición junto a la migración cold y cambia el botón para establecerlo como migración warm. Haz clic en *Guardar*. Ahora haz clic en *Iniciar* para iniciar la migración.


NOTE: Asegúrate de que, al migrar desde el almacenamiento en bloques en VMware, tengas seleccionada la clase de almacenamiento en bloques para la VM de OpenShift Virtualization. Además, la volumeMode debe estar configurada en Block y el modo de acceso debe ser ReadWriteMany (RWX), para que puedas realizar la migración en vivo de la VM más adelante.

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-001.png["1"]

Haz clic en *0 de 1 máquinas virtuales completadas*, despliega la máquina virtual y puedes ver el progreso de la migración.

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-002.png["2"]

Al cabo de un tiempo, la transferencia del disco se completa y la migración queda a la espera de pasar al estado transición. El DataVolume está en estado Paused. Vuelve al plan y haz clic en el botón *transición*.

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-003.png["3"]

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-004.png["4"]

El cuadro de diálogo muestra la hora actual. Cambia la hora para programar la transición para más tarde, o haz clic en *Establecer transición* para llevarla a cabo de inmediato.

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-005.png["5"]

Al cabo de unos segundos, el DataVolume pasa del estado Paused a ImportScheduled y luego a ImportInProgress cuando comienza la fase de transición.

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-006.png["6"]

Cuando se completa la fase de transición, el DataVolume pasa al estado Succeeded y el PVC queda vinculado.

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-007.png["7"]

El plan de migración continúa hasta completar la fase ImageConversion y, finalmente, se completa la fase VirtualMachineCreation. La máquina virtual pasa al estado Running en OpenShift Virtualization.

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-008.png["8"]
