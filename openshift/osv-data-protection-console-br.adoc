= Haz copias de seguridad y restaura máquinas virtuales usando el servicio de Backup and Recovery de NetApp Console
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Realiza copias de seguridad y restaura máquinas virtuales en Red Hat OpenShift Virtualization usando NetApp Console. Este procedimiento cubre cómo crear y gestionar operaciones de backup y restauración a través de Backup and Recovery Service mediante NetApp Console, usando un contenedor de Azure Blob Storage como destino de la copia de seguridad.

NetApp Console ofrece gestión centralizada de servicios de almacenamiento y datos tanto en entornos locales como en la nube. Ofrece control unificado, simplicidad operativa y gestión segura. Hay varios servicios de datos y funciones de gestión a los que puedes acceder desde la interfaz de usuario en link:https://console.netapp.com["console.netapp.com"^]. Para obtener todos los detalles sobre NetApp Console, consulta link:https://docs.netapp.com/us-en/console-family/["NetApp documentación de la consola"].

Esta sección se centra en el servicio de datos NetApp Backup and Recovery para cargas de trabajo de Kubernetes, específicamente máquinas virtuales de OpenShift. NetApp Backup and Recovery Service te permite descubrir, gestionar, crear y asociar políticas de protección con tus aplicaciones y máquinas virtuales de Kubernetes mediante una única interfaz. Para obtener una guía completa, consulta link:https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-kubernetes-protect-overview.html["NetApp Backup and Recovery documentación"].

== Flujo de trabajo de backup y recuperación

. *Asegúrate de tener un Console Agent*
+
Asegúrate de que tienes un agente de consola implementado en el entorno en el que se ejecuta tu clúster de OpenShift. Para obtener más información sobre cómo instalar un agente de consola, consulta link:https://docs.netapp.com/us-en/console-setup-admin/concept-agent-install-options.html#agent-console-deploy["NetApp Documentación de configuración de la consola"].

+
.Mostrar ejemplo
[%collapsible]
====
En NetApp Console, haz clic en Deploy Agent como se muestra en la captura de pantalla y despliega el agente en el entorno donde se ejecuta el clúster de OpenShift.

En este ejemplo, elegimos on-premises. Copia la URL del instalador del agente, usa el comando curl para descargar el instalador del agente en la máquina Ubuntu que se ejecuta en vSphere y ejecuta el instalador para desplegar el agente de la Console.

Regístralo utilizando la dirección IP del agente en la URL. Consulta link:https://docs.netapp.com/us-en/console-setup-admin/task-install-agent-on-prem-ova.html["la documentación"] para obtener instrucciones detalladas. Una vez registrado el agente, podrás verlo en NetApp Console, como se muestra en la captura de pantalla siguiente.

. *Configurar un contenedor de Azure Blob Storage como destino de copia de seguridad para un escenario de copia de seguridad y restauración de las máquinas virtuales de OpenShift.*
+
En este ejemplo, utilizamos Azure Blob Storage como destino de copia de seguridad. También puedes utilizar AWS S3, Google Cloud Storage, StorageGRID o ONTAP object storage como destinos de copia de seguridad.

+
Para obtener más información, consulta link:https://docs.netapp.com/us-en/data-services-backup-recovery/br-start-setup.html#prepare-the-backup-destination["NetApp documentación de la consola"].

+
Para detectar el contenedor de Azure Blob Storage como destino de copia de seguridad en NetApp Console, necesitarás la siguiente información del portal de Azure.

+
[listing]
----
Application (Client) ID:
Client Secret:
Directory (tenant) ID:
----
+
Para obtener estos valores, haz lo siguiente en el portal de Azure.

+
.. Crea un contenedor de Blob Storage.
.. Crea una aplicación empresarial y asigna los permisos de acceso necesarios para esta aplicación al contenedor de Blob Storage. Obtén el Application ID, Directory ID y Client Secret de esta aplicación para usarlos en NetApp Console y descubrir el contenedor de Azure Blob Storage como destino de la copia de seguridad.

. *En NetApp Console, descubre el clúster de OpenShift.*
+
.Mostrar ejemplo
[%collapsible]
====
Este paso requiere que el clúster de OpenShift esté en funcionamiento y que el agente de la Console esté implementado en el mismo entorno. Sigue estos pasos para descubrir el clúster de OpenShift en NetApp Console.

.. En NetApp Console, selecciona *Inventory > Kubernetes* e introduce los detalles del clúster de OpenShift.
.. Selecciona el agente implementado en el mismo entorno donde se encuentra el clúster de OpenShift.
.. Copia los comandos proporcionados para instalar Trident Protect y ejecútalos desde una máquina que se use para ejecutar  `oc` comandos en tu clúster de OpenShift.
.. Una vez instalado correctamente Trident Protect, haz clic en *Descubrir*. NetApp detecta el clúster de OpenShift y todos sus recursos a través del agente, y los muestra en la interfaz de usuario.

Las credenciales de Azure se almacenan en NetApp Console y se utilizan para acceder al almacenamiento de blobs de Azure para las operaciones de copia de seguridad y restauración.

En este paso, crea una aplicación en NetApp Console para las máquinas virtuales que necesitan ser protegidas. Puedes crear una aplicación para una sola máquina virtual o para varias máquinas virtuales. También puedes usar espacios de nombres para crear una aplicación que proteja todas las máquinas virtuales en un espacio de nombres en el clúster OpenShift.

También necesitarás una política de protección para asociar con la aplicación. Puedes crear una nueva política de protección o usar una existente. En este ejemplo, se crea una nueva política de protección con la aplicación y se asocia con ella. Se crean copias de seguridad para la aplicación según la política de protección asociada a ella. La cantidad de copias de seguridad que se retienen depende de la política de retención definida en la política de protección.

Si tu aplicación o máquinas virtuales tienen más de un disco y la copia de seguridad falla con el error que se muestra a continuación, edita las reglas de la API  `volumegroupsnapshots` y  `volumegroupsnapshotcontents` para que incluyan los verbos indicados.

.<strong>Mostrar solución</strong>
[%collapsible]
=====
image:osv-data-protection-console-043.png["Error al proteger máquinas virtuales que tienen más de un disco cada una"]

[source, yaml]
----
- apiGroups:
  - groupsnapshot.storage.k8s.io
  resources:
  - volumegroupsnapshots
  verbs:
  - get
  - list
  - watch
  - create
  - delete
  - update
  - patch
- apiGroups:
  - groupsnapshot.storage.k8s.io
  resources:
  - volumegroupsnapshotcontents
  - volumegroupsnapshotcontents/status
  verbs:
  - get
  - list
  - watch
  - create
  - delete
  - update
  - patch
- apiGroups:
  - groupsnapshot.storage.k8s.io
  resources:
  - volumegroupsnapshotclasses
  verbs:
  - get
  - list
  - watch

----
