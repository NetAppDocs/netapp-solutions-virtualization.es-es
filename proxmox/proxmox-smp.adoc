= Configurar y utilizar Storage Manager para Proxmox
:allow-uri-read: 
:icons: font

Storage Manager for Proxmox (SMP) proporciona un completo aprovisionamiento de almacenamiento y gestión del ciclo de vida para máquinas virtuales en entornos Proxmox que utilizan sistemas de almacenamiento NetApp ONTAP. SMP permite a los administradores gestionar el almacenamiento directamente, simplificando las operaciones mediante capacidades automatizadas de aprovisionamiento, copia de seguridad, restauración y clonación.



== Conoce Storage Manager para Proxmox

Storage Manager for Proxmox (SMP) ofrece una gestión integral del almacenamiento para máquinas virtuales dentro de entornos Proxmox que utilizan sistemas de almacenamiento NetApp. Al permitir a los administradores gestionar directamente el almacenamiento, SMP simplifica tanto las tareas de almacenamiento como las de gestión de datos en entornos Proxmox.



=== Funciones clave

* *Gestión integral del almacenamiento*: SMP proporciona una gestión completa del ciclo de vida de las máquinas virtuales, desde el aprovisionamiento de almacenamiento hasta las copias de seguridad y la recuperación.
* *Gestión directa del almacenamiento*: los administradores de Proxmox pueden gestionar directamente el almacenamiento, agilizando las operaciones y reduciendo la complejidad.
* *Instalador independiente*: SMP se entrega como un instalador independiente que se conecta sin problemas a los nodos Proxmox a través de API.




=== Capacidades

Cada componente de SMP está diseñado para mejorar la eficiencia de la gestión del almacenamiento:

* *Aprovisionamiento de almacenamiento*: Asigna y gestiona fácilmente recursos de almacenamiento para hosts Proxmox y máquinas virtuales asociadas.
+
** *Aprovisionamiento de almacenamiento NFS*: Configura y ajusta el almacenamiento NFS de ONTAP para usarlo con tu entorno Proxmox.
** *aprovisionamiento de almacenamiento iSCSI*: Implementa LVM grueso con multivía para una gestión del almacenamiento iSCSI robusta y eficiente.


* *Copias de seguridad y recuperación*: Realiza copias de seguridad fiables y recuperaciones rápidas para garantizar la integridad de los datos y la disponibilidad.
+
** *Backups coherentes ante fallos y coherentes a nivel de archivos*: Garantiza la integridad de los datos configurando mecanismos de backup tanto coherentes ante fallos como coherentes a nivel de archivos.
** *Operaciones de restauración de máquinas virtuales*: Ejecuta procesos de restauración de máquinas virtuales para recuperar datos y mantener la disponibilidad del sistema.


* *Eficiencia del almacenamiento*: ONTAP proporciona un conjunto completo de tecnologías de eficiencia del almacenamiento diseñadas para minimizar el almacenamiento necesario para los datos. Estas tecnologías no solo reducen el consumo de almacenamiento, sino que también mejoran el rendimiento del sistema, ofreciendo:
+
** *Aprovisionamiento rápido de máquinas virtuales*: agiliza el aprovisionamiento de máquinas virtuales para un despliegue más rápido.
** *Copias de conjuntos de datos más rápidas*: Acelera la duplicación de conjuntos de datos para una gestión eficiente de los datos.






=== Prerrequisitos

Asegúrate de haber completado lo siguiente antes de continuar:

* Cluster Proxmox Virtual Environment (PVE) 9.x o posterior en funcionamiento.
* Almacenamiento All Flash FAS (AFF) de ONTAP con los protocolos NFS e iSCSI activados en el nivel SVM. Para obtener instrucciones, consulta link:proxmox-ontap-nfs.html["Configurar almacenes de datos NFS con ONTAP"] y link:proxmox-ontap-iscsi.html["Configurar Proxmox VE 9.x para iSCSI con almacenamiento ONTAP"].
+
[NOTE]
====
Para la configuración de iSCSI LUN, se debe habilitar lo siguiente:

*Instalar multivía e iSCSI en todos los hosts Proxmox:*

[source, shell]
----
apt update
apt install open-iscsi multipath-tools -y
systemctl enable --now iscsid multipathd
----
*Configura el iniciador iSCSI y conéctate a través de iSCSI (en hosts Proxmox):*

Crear o actualizar `/etc/multipath.conf`:

[source, shell]
----
touch /etc/multipath.conf
----
O usa:

[source, shell]
----
cat << 'EOF' > /etc/multipath.conf
defaults {
   user_friendly_names yes
   find_multipaths yes
}
blacklist {
   devnode "^sd[a]$"
}
EOF
----
====
* Los hosts Proxmox y el almacenamiento ONTAP son accesibles en red desde el host SMP.

== Configura Storage Manager para Proxmox

Cuando el almacenamiento y la conectividad a los nodos de almacenamiento y Proxmox se hayan configurado correctamente, configura SMP para automatizar el aprovisionamiento y la protección de las máquinas virtuales aprovechando las funcionalidades de ONTAP Snapshot y FlexClone.

El primer paso es descubrir y añadir el punto final de almacenamiento ONTAP y luego los nodos Proxmox de destino a SMP.

.Antes de empezar
Asegúrese de que se cumplan los siguientes requisitos previos:

* SMP está instalado
* La instancia SMP tiene conectividad con los nodos Proxmox y los sistemas de almacenamiento
* Cluster de ONTAP para proporcionar pools de almacenamiento NFS o iSCSI
* La resolución DNS está en su lugar cuando se utilizan nombres DNS. De lo contrario, usa direcciones IP para el descubrimiento.
* El entorno cuenta con versiones compatibles de los servidores Proxmox Virtual Environment

== Aprovisionar almacenamiento

El aprovisionamiento de un volumen como un conjunto de almacenamiento, comúnmente conocido como datastore, crea un contenedor lógico para máquinas virtuales y sus discos virtuales (QCOW2 o RAW). SMP permite aprovisionar un datastore y luego adjuntarlo a un único host o a todos los hosts dentro de un clúster.

[cols="2*"]
|===
| Configuración de ONTAP | Valor por defecto 


| Reserva de snapshot (-percent-snapshot-space) | 0% 


| Reserva fraccionaria (-fractional-reserve) | 0% 


| Actualización del tiempo de acceso (-atime-update) | Deshabilitado 


| Readahead mínimo (-min-readahead) | Deshabilitado 


| Instantáneas programadas | Ninguno 


| Eficiencia del almacenamiento | Habilitado 


| Garantía de volumen | Ninguno (thin provisioned) 


| Tamaño automático del volumen | crecer_contraer 


| Reserva de espacio LUN | Deshabilitado 


| Asignación de espacio LUN | Habilitado 
|===


=== Comportamiento de NConnect

Cuando utilices almacenamiento NFS con Proxmox VE, ten en cuenta el comportamiento específico del cliente de NFS de Linux (tal y como lo utiliza Debian, el SO base de Proxmox VE). El kernel de Linux puede reutilizar los parámetros de montaje de la primera vez que se monta una exportación NFS en un nodo. Si se vuelve a montar la misma exportación con diferentes opciones de montaje, puede que no se apliquen esas nuevas opciones.

Como práctica recomendada, crea y monta exportaciones independientes y utiliza direcciones IP de servidores NFS diferentes, como LIF de datos distintos. SMP detecta automáticamente las conexiones existentes junto con sus valores nconnect asociados y aplica el valor nconnect adecuado a cualquier conexión nueva.



== Proteger las máquinas virtuales

Cuando las máquinas virtuales se ejecutan en almacenes de datos NFS respaldados por ONTAP, el enfoque de protección más racionalizado consiste en dejar que Proxmox establezca la coherencia del sistema de archivos, mientras que ONTAP captura el punto de backup real en la capa de almacenamiento. Las snapshots de ONTAP proporcionan copias a un momento específico rápidas y con gestión eficiente del espacio del volumen del almacén de datos y pueden utilizarse después para recuperar almacenes de datos completos, archivos individuales o contenido de máquinas virtuales.

SMP proporciona un orquestador sencillo para la protección de datos coherente con fallos y con máquinas virtuales, así como para la gestión de clones. Este software simplifica los ciclos de vida de backup, restauración y clonación mediante la creación de políticas de backup que pueden aplicarse a varios trabajos. Estas políticas pueden definir la programación, la retención, la replicación y otras capacidades. Permiten la selección opcional de snapshots coherentes con archivos; este enfoque utiliza la capacidad de QEMU para poner en reposo la E/S antes de tomar un snapshot de ONTAP.

=== Detalles del flujo de trabajo de copia de backup

El flujo de trabajo de copia de seguridad a grandes rasgos incluye:

. *Define el ámbito de protección*: Selecciona si el objetivo de protección es un datastore (volumen ONTAP), un grupo de recursos (múltiples volúmenes o VMs) o una VM individual.
. *Confirma el método de coherencia*: Elige cómo se crea la coherencia antes de la snapshot de ONTAP:
+
** Congelación/descongelación del agente invitado de QEMU para la inactividad del sistema de archivos dentro del invitado y la integración con Windows VSS.


. *Vuelca los archivos de configuración de la máquina virtual*.
. *Toma la snapshot de ONTAP*: Captura una snapshot de ONTAP del volumen del datastore mientras las máquinas virtuales están en un estado coherente. Esto se convierte en la copia de backup retenida.
. *Elimina las instantáneas temporales de Proxmox*: Elimina las instantáneas de Proxmox después de que se complete la instantánea de ONTAP para evitar el crecimiento del archivo delta.
. *Aplicar políticas de retención y replicación (mediante políticas)*: aplica la política de retención de ONTAP configurada y replica el conjunto de snapshots si está activada la replicación de recuperación ante desastres.


Este enfoque minimiza el espacio de almacenamiento y el ancho de banda de la red almacenando únicamente la información modificada. Usa copias Snapshot para hacer copias rápidas de la máquina virtual o del almacén de datos sin afectar el rendimiento y luego envíalas a un sistema secundario usando la tecnología NetApp SnapMirror para la protección de datos a largo plazo y fuera del sitio.

=== Detalles del flujo de trabajo de restauración

El flujo de trabajo de restauración varía en función del tipo de almacenamiento subyacente e implica los siguientes pasos clave:

. *Define el alcance de la restauración*: selecciona si el objetivo de protección es un datastore (volumen ONTAP), un grupo de recursos (múltiples volúmenes o VMs) o una VM individual.
. *Ejecuta la operación de restauración*: El proceso de restauración de la máquina virtual varía en función del protocolo de almacenamiento:
+
** *Restauraciones basadas en NFS*: Al restaurar un volumen o un almacén de datos en NFS, SMP inicia una operación de restauración de instantánea de volumen en el volumen de destino. Para las máquinas virtuales alojadas en volúmenes NFS, SMP valida los parámetros de restauración, incluidos el ID de la máquina virtual, el nombre de la instantánea, los ID de sitio y el tipo de almacenamiento. Si la máquina virtual está en ejecución, SMP la detiene sondeando su estado y emitiendo un comando de parada. A continuación, el sistema restaura los discos de la máquina virtual a partir de la instantánea mediante una operación de restauración a nivel de archivo.
** *restauraciones basadas en iSCSI*: Para los datastores o LUN iSCSI, SMP restaura el LUN de la instantánea, verifica el dispositivo restaurado, realiza una nueva exploración iSCSI en los nodos Proxmox y garantiza la accesibilidad del dispositivo. Para las VMs que residen en almacenamiento iSCSI, SMP clona los LUN de la instantánea, asigna los LUN clonados al host Proxmox y realiza la detección iSCSI para importarlos. Luego se añade el almacenamiento restaurado y se actualizan las configuraciones de las VMs. Los discos de las VMs se mueven de nuevo a su almacenamiento original y se realiza un seguimiento del progreso por VM. El proceso concluye con actividades de limpieza, incluida la eliminación del almacenamiento temporal y el borrado de los LUN clonados.

=== Detalles del flujo de trabajo de clonación

SMP automatiza la clonación rápida de máquinas virtuales descargando el trabajo pesado a ONTAP para realizar clones del lado del almacenamiento en lugar de copias de archivos del lado del host. De forma encubierta, SMP aprovecha la semántica de ONTAP FlexClone (clones basados en instantáneas, con capacidad de escritura y con gestión eficiente del espacio) para que los clones se creen rápidamente y consuman poca capacidad inicialmente; evita las lecturas/escrituras de datos a través del host Proxmox siempre que sea posible.

Este enfoque acelera el aprovisionamiento de máquinas virtuales al trasladar el trabajo de clonación a ONTAP, entregando clones rápidos, finos y con gestión eficiente del espacio, adecuados para Dev/Test, laboratorios y ciclos de actualización rápidos.

Por defecto, el clon se realiza contra el sistema de archivos activo. El flujo de trabajo de clonación también permite seleccionar una instantánea, en cuyo caso los clones de archivo se realizan utilizando un nuevo volumen creado a partir de la instantánea seleccionada.

== Resumen

Storage Manager para Proxmox ofrece una forma racionalizada y unificada de gestionar las operaciones de protección, clonación y recuperación en todos los entornos Proxmox. Al integrarse directamente con ONTAP, la solución aporta la eficiencia, fiabilidad y rendimiento del almacenamiento para la gran empresa a los flujos de trabajo cotidianos de las máquinas virtuales. Las instantáneas con gestión eficiente del espacio de ONTAP, las capacidades de clonación rápida y la replicación incorporada proporcionan una protección de datos coherente y predecible, mientras que SMP simplifica estas capacidades en una interfaz intuitiva y centrada en las tareas.
